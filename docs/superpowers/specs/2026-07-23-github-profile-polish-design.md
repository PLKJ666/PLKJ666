# GitHub 个人主页展示优化设计

## 目标

让未登录访客在首屏快速识别 Nike 的求职身份、业务价值和可验证交付能力，同时减少重复信息并保持真实、可防守的项目表达。

## 改动范围

1. 将 GitHub 账户公开姓名设置为 `Nike`。
2. 将置顶仓库调整为：
   - `contentguard-ai`
   - `xiaohongshu-ai-content-studio`
   - `enterprise-content-workflow`
3. 将 Profile README 首屏压缩为：姓名、职业方向、一句话价值、三项证据和四个导航入口。

## 首屏文案

```markdown
# Nike

### AI 应用开发 · 企业工作流 · 全栈后端

把内容生产、企业自动化和营销数据中的人工流程，做成真正可运行、可验证、可维护的 AI 系统。

`3 个公开项目` · `2 个真实上线系统` · `从需求到部署独立交付`

[代表项目] · [交付能力] · [真实案例] · [English]
```

## 保留边界

- 不改动公开项目的事实、测试数量、真实运行指标和脱敏声明。
- 不改动仓库可见性、仓库内容或项目代码。
- 不虚构客户、收入、效率或模型能力。
- 用户已经自行更新头像，本任务不再修改头像。

## 验收标准

- 未登录访客页面显示公开姓名 `Nike`。
- Profile README 首屏与本设计文案一致，且后续项目内容保持完整。
- 三个置顶仓库均存在，顺序与设计一致。
- Profile 仓库本地与远端分支一致。
- 通过公开 GitHub 页面重新检查姓名、README 和置顶仓库。
